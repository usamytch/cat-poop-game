## 🎮 Механики игры

### Основной цикл

- Управляй котом по комнате, уворачиваясь от хозяина и препятствий
- Шкала **срочности** постоянно растёт — при 100% случается **авария 💩**
- Хозяин преследует кота — при поимке теряется жизнь
- Стреляй какашками в хозяина, чтобы его остановить
- Собирай бонусы на карте
- Доберись до **лотка 🐾** — и переходи на следующий уровень

### Шкала срочности

Растёт каждый кадр: `urgeRate/60 * (1 + (level-1)*0.08)` с мягким капом ×1.8 (~уровень 11).
- Паника при `urge/maxUrge > 0.75`
- Авария при `urge >= maxUrge`

### Комбо-система

3 попадания в хозяина подряд = **COMBO!** + автоматическая заморозка хозяина + звук.
Каждое попадание оставляет 💩 на лице хозяина. После бегства — очищаются.

### Снижение срочности при стрельбе

| Режим | За выстрел (`shootUrgeReduce`) | За попадание (`hitUrgeReduce`) |
|---|---|---|
| 😸 Лёгкий | 1.5 | 3.0 |
| 😼 Нормал | 1.0 | 2.0 |
| 😈 Хаос | 0 | 1.5 |

`hitUrgeReduce` всегда меньше прироста за кулдаун — нельзя читерить стрельбой.

## 🗺️ Локации и препятствия

5 обычных локаций + 1 закрытая:
- **Зал** — шкафы, кресла, фикусы
- **Ванная** — раковины, унитазы, корзины
- **Кухня** — столы, холодильники, табуреты
- **Двор** — деревья, лавки, кусты
- **Дача** — поленницы, бочки, кресла-качалки
- **🏚️ Подвал** *(закрытая)* — стены-лабиринта, трубы, ящики, цепи

С 5 уровня препятствия начинают **двигаться**.

### Паттерны пола и стен

| Локация | Пол | Стена |
|---|---|---|
| 🛋️ Зал | Паркет ёлочкой (диагонали 20×20px) | Обои с ромбами |
| 🚿 Ванная | Шахматная плитка (30×30px) | Кафель 60×60px |
| 🍳 Кухня | Горизонтальные доски | Кафель-фартук 80×40px |
| 🌿 Двор | Трава с вариациями (noise-based) | Полосы неба |
| 🏡 Дача | Деревянные доски с сучками | Вагонка 32px |
| 🏚️ Подвал | Каменная плитка 40×40px с трещинами | Кирпич 60×20px с сыростью |

Все паттерны рисуются с `globalAlpha=0.07–0.10` — едва заметная фактура.

## 🏚️ Подвал — закрытая локация

Появляется случайно начиная с уровня 9. Чит-код **Shift+B** (EN) — мгновенный переход.

### Режимы лабиринта

| Режим | Уровень | Вероятность | Описание |
|---|---|---|---|
| **Коридорный** | 9+ | 25% | Горизонтальные и вертикальные стены с проходами |
| **DFS-лабиринт** | 20+ | 33% | Recursive Backtracker — единственный путь к лотку |

Пороги и вероятности: константа `BASEMENT` в [`js/config.js`](js/config.js).

### Визуальные особенности

- **🪣 Тёмный лоток** — `#1e1510` корпус, `#3a2e1e` наполнитель, пятна плесени
- **🔦 Фонарик хозяина** — конус тёплого жёлтого света (~40°, 220px) по направлению движения
- **👁️ Светящиеся глаза кота** — жёлто-зелёные, редкое моргание (~раз в 3.5 сек)

### Вмурованные предметы в стенах

Чисто визуальный декор, без коллизий. Количество: 4–7 за уровень (`BASEMENT.wallEmbedCount`).

| Предмет | Описание |
|---|---|
| 🐟 Рыбьи кости | Рыбий скелет с позвоночником и рёбрами |
| 🐭 Тряпочная мышь | Серая мышка с ушками и хвостиком |
| 🧸 Плюшевый мишка | Коричневый мишка с ушками и мордочкой |
| 🚗 Игрушечная машинка | Красная машинка с колёсами (вид сбоку) |
| ✈️ Игрушечный самолётик | Синий самолётик с крыльями (вид сверху) |
| 🥫 Банка от сока | Оранжевая цилиндрическая банка |

### Гарантии проходимости

- **Коридорный режим**: боковые полосы (col 0 и col GRID_COLS-1) **полностью свободны**
- **DFS-режим**: колонки вне сетки лабиринта (cols 28–29) — сплошные кирпичные стены; спавн кота в cols 0–26
- **Гарантия пути**: `_ensureBasementReachable()` проверяет A*-путь от спавна до лотка; если ящик/бочка заблокировали — декоративные препятствия удаляются по одному
- **Хозяин в лабиринте**: A* использует размер кота (36×36px) вместо своего (36×52px) — корректно проходит коридоры шириной 2 ячейки (80px)
- **Безопасный спавн хозяина**: spiral search от ячеечных углов сетки — не попадает в заблокированные колонки DFS (cols 28–29)

## 🔢 Сеточная генерация уровней (Grid System)

```
Игровая зона: 1200 × 600 px → 30 колонок × 15 строк = 450 ячеек
GRID = 40px
```

- **Препятствия** — размер кратен 40px (2×2, 2×4, 4×2, 4×4 ячейки), позиция выровнена по сетке
- **Нет наложений** — `occupiedCells` Set гарантирует уникальность ячеек
- **Лоток** — всегда на свободных ячейках
- **Бонусы** — спавнятся в центрах свободных ячеек
- **Детерминированность** — `level + score + globalSeed` → одинаковая карта (LCG RNG)
- **Уникальность** — `globalSeed = Date.now()` при старте → уникальный запуск
- **Проходимость щелей** — кот (36×36px) < ячейка (40px) → любая щель в 1 ячейку проходима

### Умная генерация — Zone Bias

| Зона | Объекты | Описание |
|---|---|---|
| `wall` | шкафы, комоды, раковины, холодильники | Предпочитают края |
| `corner` | кресла, корзины, бочки, ящики | Предпочитают углы |
| `center` | столы, лавки | Могут стоять в центре |
| `any` | табуреты, деревья, кусты | Без предпочтений |

С вероятностью 45–80% объект размещается в своей зоне.

### Прогрессия плотности

| Фаза | Уровни | Зазор | Центр | Кластеры |
|---|---|---|---|---|
| 🌿 Просторно | 1–3 | 1 ячейка | Свободен (70%) | Редкие |
| 🏠 Обжитое | 4–7 | 1 ячейка | Частично занят | Умеренные |
| 🗑️ Захламлено | 8+ | 0 (вплотную) | Плотный | Густые |

**Value Noise** — seeded hash без зависимостей: карта «плотности» создаёт естественные кластеры.

## 🤖 A* навигация хозяина

Хозяин использует **алгоритм A\*** по сетке для обхода препятствий:

- Путь пересчитывается каждые **~0.5 сек** на открытых уровнях, **~0.25 сек в подвале**
- **Steering corridor model** — вместо прицеливания в точный центр ячейки хозяин движется вдоль оси коридорного сегмента:
  1. `_compressToSegments(path)` — сжимает A* путь в сегменты: последовательные ячейки с одинаковым направлением объединяются в один сегмент `{startPx, endPx, dir}`
  2. `_getSteeringTarget()` — проецирует центр хозяина на ось активного сегмента, выдвигает цель на `LOOKAHEAD = GRID * 0.8 = 32px` вперёд, зажимает до конца сегмента. Проекция зажата снизу в 0 (`max(t, 0)`) — если хозяин позади startPx, цель всё равно выдвигается вперёд
  3. **Segment completion** — сегмент засчитывается когда `progress >= segLen - EPSILON` (прогресс вдоль оси): `EPSILON = GRID * 0.4 = 16px` в подвале, `8px` на открытых уровнях
  4. Когда сегменты исчерпаны — `pathTimer = 0` → немедленный пересчёт A*
- **Совместимость с wall-sliding**: steering target — точка на оси коридора, а не точный центр ячейки. Физическое скольжение вдоль стены не мешает прогрессу вдоль оси сегмента
- **Path Smoothing** (только в подвале): `_hasLineOfSight()` (Bresenham по сетке) + `_smoothPath()` — пропускает промежуточные waypoints при прямой видимости **только на прямых участках**; на поворотах smoothing отключён
- **Анти-застревание (net-displacement)**: вместо per-frame delta используется net-displacement за `CHECK_INTERVAL` кадров. Это обнаруживает **осцилляцию** (движение туда-обратно), которая сбрасывает per-frame счётчик каждый кадр, но не даёт реального прогресса. Параметры: в подвале `CHECK_INTERVAL=10`, `NET_THRESHOLD2=2px²`, `MAX_STUCK_CHECKS=2`; на открытых уровнях `CHECK_INTERVAL=15`, `NET_THRESHOLD2=4px²`, `MAX_STUCK_CHECKS=3`. После `MAX_STUCK_CHECKS` последовательных интервалов без прогресса → `pathSegments=[], segmentIndex=0, pathTimer=0` (форсированный пересчёт A*), `stuckTimer=0`. **Важно**: `path` НЕ очищается — очистка `path=[]` при `pathTimer=0` создаёт двойной триггер (`path.length===0` + `pathTimer<=0`) и бесконечный цикл пересчётов. Случайный «толчок» (`stuckNudge`) удалён — steering устраняет угловые deadlock'и
- Если A* не находит путь — немедленный повторный пересчёт
- **Footprint для A* в подвале**: используется размер кота (36×36px) вместо хозяина (36×52px) — коридоры шириной 2 ячейки (80px) физически вмещают хозяина, но `canPass` с 36×52 ложно видит коллизию со стеной
- **Почему EPSILON = GRID*0.4 в подвале**: хозяин 36×52px центрируется на ячейке 40px → его центр физически не может достичь точного `cellToPixelCenter` конечной ячейки (стена блокирует на ~6px раньше по перпендикулярной оси). Порог 16px > 6px с запасом, но меньше GRID/2=20px — не срабатывает на коротких сегментах (1 ячейка = 40px). Lateral snap был удалён: он создавал deadlock на поворотах (snap тянул хозяина перпендикулярно, но стена предыдущего коридора блокировала)
- **`_debugSteering` — глобальный флаг**: объявлен как `var _debugSteering = false` в [`js/renderer.js`](js/renderer.js) (не `let`!) — `var` добавляет переменную в `window`, делая её доступной из [`js/owner.js`](js/owner.js). `let` в браузерных скриптах НЕ добавляется в `window` и недоступен из других скриптов. Переключается клавишей **Shift+G** в игре. Включает логи `[REPATH]`, `[STEER]`, `[STUCK]`, `[ESCAPE]`, `[STEER-ZERO]` в консоли

### Поведение хозяина — «человечность» (только на открытых уровнях)

- **Дрейф** — случайное угловое отклонение ±10°, меняется каждые ~1.5 сек
- **Микро-заморозка** — ~раз в 4 сек замирает на 0.2 сек
- **Реакция на выстрел** — немедленный пересчёт пути

> В подвале дрейф и микро-заморозки **отключены** — в коридорах они вызывали застревание у стен.

## 😱 Паника-режим

При срочности > 75%:
- Шкала мигает красным с пульсирующей рамкой
- Лоток светится красным
- Экран трясётся (кот дрожит)
- Тревожный звук каждые полсекунды
- **Мелодия** переключается на паника-версию (185 BPM vs 138, зеркальный бас, тремоло)
- **Виньетка** — тёмные края экрана (0.35→0.70 alpha)
- **Красные вспышки** — периодические импульсы, частота и яркость растут к 100%
- **Хроматическая аберрация** — красно-синие полосы по краям
- **Wave distortion** — синусоидальный skew всего экрана

Все эффекты через Canvas 2D без `ctx.filter` — стабильные 60fps.

## 🔑 Ключевые инварианты для агентов

- Шкала срочности: `urgeRate/60 * (1 + (level-1)*0.08)`, кап ×1.8
- Паника при `urge/maxUrge > 0.75`, авария при `urge >= maxUrge`
- Комбо при 3 попаданиях подряд → `owner.flee()`
- `hitUrgeReduce` всегда меньше прироста за кулдаун
- Только Vanilla JS — никаких фреймворков и npm-пакетов в рантайме
- Весь звук — процедурный через Web Audio API
- Вся графика — Canvas 2D
- Прогресс — только через `localStorage`
- Запуск: открыть `index.html` в браузере (или `python3 -m http.server 8080`)

# 🐱 Cat Poop Game — Бэнгер Edition

> Доведи кота до лотка — пока не поздно!

Браузерная аркада на чистом JavaScript + Canvas 2D. Кот хочет в туалет. Хозяин не даёт. Шкала срочности заполняется. Успей добраться до лотка.

---

## 🎮 Геймплей

- Управляй котом по комнате, уворачиваясь от хозяина и препятствий
- Шкала **срочности** постоянно растёт — при 100% случается **авария 💩**
- Хозяин преследует кота — при поимке игра заканчивается
- Стреляй какашками в хозяина, чтобы его остановить
- Собирай бонусы на карте
- Доберись до **лотка 🐾** — и переходи на следующий уровень

---

## 🕹️ Управление

| Клавиша | Действие |
|---|---|
| `WASD` / `←↑↓→` | Движение кота |
| `Пробел` / `X` | Выстрел какашкой в направлении движения |
| `1` / `2` / `3` | Выбор сложности на стартовом экране |
| `Enter` | Старт / Возврат в меню |
| `M` | Вкл/выкл звук |

## ✨ Фичи

### 🔊 Звуки (Web Audio API)
Все звуки генерируются процедурно через осцилляторы — никаких внешних файлов:
- Мяу при старте, пуканье при выстреле, попадание, тревога, победа, провал, комбо, подбор бонуса

### 😱 Паника-режим
При срочности > 75%:
- Шкала мигает красным с пульсирующей рамкой
- Лоток светится красным
- Экран трясётся
- Тревожный звук каждые полсекунды
- **Мелодия переключается на паника-версию** — ускоренная (185 BPM vs 138), зеркальный бас, тремоло-слой

Когда срочность снижается обратно ниже 75% (например, таблетка 💊), мелодия возвращается к обычной.

### 🎉 Оверлеи
- **Победа** — конфетти из 💩⭐✨🎉 с физикой
- **Авария** — лужа 💧 с брызгами
- **Поимка** — тёмный оверлей с анимацией

### 🎁 Бонусы
| Бонус | Эффект | Длительность |
|---|---|---|
| 🐟 Рыбка | Ускорение кота x1.7 | 5 сек |
| 🧶 Клубок | Хозяин замирает | 5 сек |
| 💊 Таблетка | -30% срочности | мгновенно |

### 💥 Комбо-система и какашки на лице
3 попадания в хозяина подряд = **COMBO!** + автоматическая заморозка хозяина + звук

Каждое попадание оставляет 💩 на лице хозяина — они накапливаются и видны во время бегства.
После того как хозяин вернулся из угла (флаг бегства истёк), все какашки с лица исчезают.

### 🎯 Снижение срочности при стрельбе
Стрельба какашками снижает шкалу срочности кота двумя способами:

**1. При каждом выстреле** (`DIFF[difficulty].shootUrgeReduce`) — на лёгком и нормале срочность снижается сразу при выстреле, даже если промазал:

| Режим | Снижение за выстрел |
|---|---|
| 😸 Лёгкий | 1.5 |
| 😼 Нормал | 1.0 |
| 😈 Хаос | 0 (нет снижения) |

**2. При попадании в хозяина** (`DIFF[difficulty].hitUrgeReduce`) — дополнительное снижение при прямом попадании:

| Режим | Снижение за попадание |
|---|---|
| 😸 Лёгкий | 3.0 |
| 😼 Нормал | 2.0 |
| 😈 Хаос | 1.5 |

На хаосе снижение срочности только при попадании — промахи не помогают.

### 😈 Режимы сложности
| Режим | Срочность | Хозяин | Появляется |
|---|---|---|---|
| 😸 Лёгкий | Медленная | Медленный | Со 2 уровня |
| 😼 Нормал | Стандарт | Средний | С 1 уровня |
| 😈 Хаос | Быстрая | Быстрый | С 1 уровня |

### 🏆 Прогресс
- Рекорд и статистика сохраняются в `localStorage`
- Отображаются на стартовом экране: рекорд, лучший уровень, поймано хозяином, аварий, какашек выпущено

### 🗺️ Локации и препятствия
5 уникальных локаций с палитрами и декорациями:
- **Зал** — шкафы, кресла, фикусы
- **Ванная** — раковины, унитазы, корзины
- **Кухня** — столы, холодильники, табуреты
- **Двор** — деревья, лавки, кусты
- **Загородный дом** — поленницы, бочки, кресла-качалки

С 5 уровня препятствия начинают **двигаться**.

### 🎯 Случайный спавн в углах экрана

На старте каждого уровня кот и хозяин всегда появляются **в разных углах экрана**:
- **Кот** — спавнится в случайном из 4 углов игровой зоны (детерминировано через seeded RNG)
- **Хозяин** — автоматически выбирает **самый дальний угол** от кота

Это гарантирует максимальное расстояние между ними в начале каждого уровня и делает каждый старт уникальным.

### 🔢 Сеточная генерация уровней (Grid System)

Все препятствия, лоток и бонусы расставляются по **чёткой сетке 40×40 пикселей**:

```
Игровая зона: 1120 × 520 px → 28 колонок × 13 строк = 364 ячейки
```

- **Препятствия** — размер кратен 40px (2×2, 2×4, 4×2, 4×4 ячейки), позиция всегда выровнена по сетке
- **Нет наложений** — `occupiedCells` Set гарантирует, что ни одно препятствие не перекрывает другое
- **Лоток** — всегда на свободных ячейках, никогда не появляется на препятствии
- **Бонусы** — спавнятся в центрах свободных ячеек
- **Декор** — фоновые элементы (ковры, камни, заплатки) рисуются по сетке, но не блокируют движение
- **Детерминированность внутри сессии** — та же пара `level + score + globalSeed` всегда даёт одинаковую карту (LCG RNG)
- **Уникальность каждого запуска** — `globalSeed` берётся из `Date.now()` при старте игры, поэтому каждый новый запуск даёт уникальное расположение препятствий, лотка и бонусов
- **Гарантированная проходимость щелей** — кот (36×36px) и хозяин (36×52px) меньше одной ячейки сетки (40px), поэтому любая щель в 1 ячейку между препятствиями гарантированно проходима для обоих

### 🤖 A* навигация хозяина

Хозяин использует **алгоритм A\*** по сетке для обхода препятствий:
- Путь пересчитывается каждые ~0.5 сек или при исчерпании текущего
- Хозяин плавно следует по waypoint'ам ячеек, скользя вдоль стен
- **Анти-застревание**: если хозяин не двигается >0.5 сек — форсируется пересчёт пути и добавляется случайный «толчок» в свободном направлении
- Если A* не находит путь — немедленный повторный пересчёт (без ожидания таймера)

### 🧠 Поведение хозяина — «человечность»

Хозяин ведёт себя не как робот:
- **Дрейф** — небольшое случайное угловое отклонение (±10°) при движении, меняется каждые ~1.5 сек
- **Микро-заморозка** — ~раз в 4 сек хозяин замирает на 0.2 сек (задумался)
- **Реакция на выстрел** — при выстреле кота хозяин немедленно пересчитывает путь

### 💬 Визуальные индикаторы хозяина

Над головой хозяина отображается знак, отражающий его состояние:
| Знак | Состояние |
|---|---|
| `!` красный | Преследует кота — знает куда бежать |
| `‼` оранжевый | Услышал выстрел — реагирует |
| `?` жёлтый | Потерялся / тупит / задумался |
| *(нет знака)* | Убегает после комбо (мигает) |

## ⚡ Производительность

Игра оптимизирована для стабильных 60 fps в любом браузере:

| Оптимизация | Файл | Эффект |
|---|---|---|
| **A\* min-heap** — O(log n) вместо O(n) поиска | [`js/pathfinding.js`](js/pathfinding.js) | 5–10× ускорение навигации хозяина |
| **Целочисленный `cellKey`** — число вместо строки | [`js/level.js`](js/level.js) | 3–5× быстрее Map/Set операции |
| **Кэш `getPlayBounds()`** — синглтон вместо new | [`js/utils.js`](js/utils.js) | Нет GC-давления от 15+ аллокаций/кадр |
| **Offscreen canvas** — фон/декор/статичные препятствия | [`js/renderer-bg.js`](js/renderer-bg.js) | 40–60% меньше работы рендерера |
| **Emoji-кэш** — `drawImage` вместо `fillText` | [`js/particles.js`](js/particles.js), [`js/projectiles.js`](js/projectiles.js), [`js/bonuses.js`](js/bonuses.js) | 3–5× быстрее рендеринг частиц |
| **`Date.now()` один раз за кадр** | [`js/renderer.js`](js/renderer.js) | Нет повторных системных вызовов |
| **Swap-and-pop** — O(1) вместо O(n) `splice` | [`js/particles.js`](js/particles.js), [`js/projectiles.js`](js/projectiles.js) | Нет сдвига массивов при удалении |
| **Переиспользование Map/Set в A\*** — `.clear()` | [`js/pathfinding.js`](js/pathfinding.js) | Нет аллокаций каждые 30 кадров |
| **Squared distance** — без `Math.sqrt` где не нужен | [`js/owner.js`](js/owner.js) | Меньше вычислений в горячих путях |
| **Melody `setTimeout`** — вместо rAF | [`js/audio.js`](js/audio.js) | Освобождает rAF бюджет для рендеринга |
| **`setFont()` кэш** — не парсить font-строку повторно | [`js/utils.js`](js/utils.js) | Нет лишнего парсинга CSS-шрифта |
| **`WORLD` константы** — вместо `canvas.width/height` | [`js/renderer-bg.js`](js/renderer-bg.js), [`js/renderer-hud.js`](js/renderer-hud.js) | Нет DOM-обращений в горячих путях |

## 🏗️ Технологии

- **Vanilla JS** — никаких фреймворков
- **Canvas 2D API** — вся графика
- **Web Audio API** — процедурный звук
- **requestAnimationFrame** — игровой цикл
- **localStorage** — сохранение прогресса
- **LCG Seeded RNG** — детерминированная генерация уровней
- **Vitest** — unit- и интеграционные тесты (Node.js, без браузера)

## 🚀 Запуск

Открой `index.html` в браузере — и играй. Никакой сборки, никаких зависимостей.

```bash
# Вариант через локальный сервер (рекомендуется для звука)
python3 -m http.server 8080
# затем открой http://localhost:8080
```

> **Примечание:** Web Audio API требует пользовательского взаимодействия для инициализации. Звук заработает после первого нажатия клавиши.
